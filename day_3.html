Day 3: Inheritance & Polymorphism

Week 1: OOP Foundations

📚 Theory Topics

Inheritance

Inheritance allows a new class (derived or child class) to inherit properties and methods from an existing class (base or parent class). This promotes code reuse. The classic example is a `Dog` class inheriting from an `Animal` class. The `Dog` automatically gets the `Animal`'s properties like `Age` and methods like `Eat()`.

Polymorphism

Meaning "many forms," polymorphism allows objects of different classes to be treated as objects of a common parent class. It's often achieved through `virtual` methods in a base class and `override` methods in a child class. For example, you can have a list of `Shape` objects, where some are `Circle`s and others are `Rectangle`s. When you call the `Draw()` method on each shape, the correct version of `Draw()` (the circle or the rectangle version) is executed automatically.

💻 Coding Practice

Create a `Shape` base class and derive `Circle` and `Rectangle` from it.

public abstract class Shape
{
    // An abstract method has no body and MUST be implemented by derived classes
    public abstract double CalculateArea();
}

public class Circle : Shape
{
    public double Radius { get; set; }
    public override double CalculateArea()
    {
        return Math.PI * Radius * Radius;
    }
}

public class Rectangle : Shape
{
    public double Width { get; set; }
    public double Height { get; set; }
    public override double CalculateArea()
    {
        return Width * Height;
    }
}
