Day 2: Constructors, Static Members & Object Lifecycle

Week 1: OOP Foundations

📚 Theory Topics

Constructors

A constructor is a special method for creating and initializing an object instance of a class. It's called automatically when you use the `new` keyword. A class can have multiple constructors (overloading), including a parameterless default constructor.

Static Members

Static members (fields, methods, properties) belong to the class itself, not to any specific object instance. There is only one copy of a static member, shared across all objects of the class. You access them using the class name, e.g., `Math.PI`. They are useful for utility functions or shared data (like a counter).

Object Lifecycle

An object's life in .NET is managed by the Common Language Runtime (CLR). It begins with **creation** (`new`), followed by **usage** (calling methods, accessing properties), and ends with **destruction**. Destruction is handled by the Garbage Collector (GC), which automatically reclaims memory from objects that are no longer reachable.

💻 Coding Practice

Create a `Logger` class using a static instance (a simple Singleton approach).

public class Logger
{
    // A private static instance of the Logger itself
    private static Logger _instance;

    // Private constructor prevents direct instantiation
    private Logger() { }

    // Public static method to get the single instance
    public static Logger GetInstance()
    {
        if (_instance == null)
        {
            _instance = new Logger();
        }
        return _instance;
    }

    public void Log(string message)
    {
        Console.WriteLine($"[{DateTime.Now}]: {message}");
    }
}
